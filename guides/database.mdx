## Overview

MEME uses a local SQLite database to persist every pipeline run. The database file is created automatically on first run at:

```
<project_root>/meme_history.db
```

This path is set in `data_manager/db.py`:

```python
# data_manager/db.py
DB_PATH = os.path.join(os.path.dirname(os.path.dirname(__file__)), "meme_history.db")
```

The database is initialized by `init_db()`, which is called at the top of `run_full_pipeline()` before any processing begins. It creates all tables and indexes if they do not already exist, so it is safe to call on every run.

## Table reference

### `daily_runs` — run control

Stores one row per executed run date. Its primary purpose is to prevent the pipeline from re-running for the same date.

```sql
CREATE TABLE IF NOT EXISTS daily_runs (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    run_date        TEXT NOT NULL UNIQUE,
    universe_size   INTEGER NOT NULL,
    total_arguments INTEGER NOT NULL,
    k_clusters      INTEGER NOT NULL,
    created_at      TEXT NOT NULL DEFAULT (datetime('now'))
);
```

| Column | Description |
|---|---|
| `run_date` | ISO date string (`YYYY-MM-DD`). Unique constraint prevents duplicate runs. |
| `universe_size` | Number of tickers processed in this run. |
| `total_arguments` | Total valid investment arguments generated across all tickers. |
| `k_clusters` | The K value used for the GMM in this run. |
| `created_at` | Wall-clock timestamp of when the record was inserted. |

### `arguments` — investment arguments

Stores every structured investment argument generated by the LLM generator agent, along with its cluster assignment and GMM probability.

```sql
CREATE TABLE IF NOT EXISTS arguments (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    run_date        TEXT NOT NULL,
    ticker          TEXT NOT NULL,
    cluster_id      INTEGER,
    polarity        INTEGER NOT NULL,
    justification   TEXT NOT NULL,
    evidence        TEXT,
    probability     REAL,
    embedding_text  TEXT,
    FOREIGN KEY (run_date) REFERENCES daily_runs(run_date)
);
```

| Column | Description |
|---|---|
| `ticker` | Stock symbol (e.g., `AAPL`). |
| `cluster_id` | Index of the GMM cluster with the highest posterior probability. |
| `polarity` | `1` = bullish, `-1` = bearish. |
| `justification` | The LLM's analytical justification for the argument. |
| `evidence` | The empirical evidence cited by the LLM. |
| `probability` | The GMM posterior probability for the assigned cluster (0.0–1.0). |
| `embedding_text` | The text string that was sent to the embedding model (not the vector itself). |

### `narratives` — cluster centroids and momentum

Stores one row per cluster per run date. Centroids are the GMM component means (stored as JSON), and `ema_momentum` is updated after returns are computed in Phase 3.

```sql
CREATE TABLE IF NOT EXISTS narratives (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    run_date        TEXT NOT NULL,
    cluster_id      INTEGER NOT NULL,
    centroid_json   TEXT NOT NULL,
    avg_return      REAL DEFAULT 0.0,
    ema_momentum    REAL DEFAULT 0.0,
    n_arguments     INTEGER DEFAULT 0,
    FOREIGN KEY (run_date) REFERENCES daily_runs(run_date),
    UNIQUE(run_date, cluster_id)
);
```

| Column | Description |
|---|---|
| `cluster_id` | Narrative index (0 to K-1). |
| `centroid_json` | JSON array of the GMM centroid vector for this cluster. Used by the temporal alignment algorithm on the next run. |
| `avg_return` | Average close-to-close return of tickers assigned to this cluster on this date. |
| `ema_momentum` | EMA-smoothed momentum of `avg_return` across days (α=0.2). |
| `n_arguments` | Number of arguments assigned to this cluster on this date. |

### `portfolio_history` — long/short positions

Stores every position in the LONG and SHORT portfolios for each run.

```sql
CREATE TABLE IF NOT EXISTS portfolio_history (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    run_date        TEXT NOT NULL,
    ticker          TEXT NOT NULL,
    side            TEXT NOT NULL CHECK(side IN ('LONG', 'SHORT')),
    rank            INTEGER NOT NULL,
    signal          REAL NOT NULL,
    weight          REAL NOT NULL,
    created_at      TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (run_date) REFERENCES daily_runs(run_date)
);
```

| Column | Description |
|---|---|
| `side` | `LONG` or `SHORT`. |
| `rank` | Position's rank within its leg (1 = strongest signal). |
| `signal` | The composite score from `score_tickers()`. |
| `weight` | Equal-weight allocation within the leg (e.g., `0.333` for 3 positions). |

## Querying the database

Open the SQLite CLI from the project root:

```bash
sqlite3 meme_history.db
```

### View all run dates

```sql
SELECT run_date, universe_size, total_arguments, k_clusters, created_at
FROM daily_runs
ORDER BY run_date DESC;
```

### Read the latest portfolio

```sql
SELECT * FROM portfolio_history ORDER BY run_date DESC;
```

### Read LONG positions only for a specific date

```sql
SELECT ticker, rank, signal, weight
FROM portfolio_history
WHERE run_date = '2025-01-15' AND side = 'LONG'
ORDER BY rank ASC;
```

### Read narrative momentum history

```sql
SELECT run_date, cluster_id, ema_momentum
FROM narratives
ORDER BY run_date DESC, cluster_id ASC;
```

### Inspect arguments for a single ticker

```sql
SELECT run_date, cluster_id, polarity, probability, justification
FROM arguments
WHERE ticker = 'AAPL'
ORDER BY run_date DESC;
```

### Count arguments per narrative per day

```sql
SELECT run_date, cluster_id, n_arguments, avg_return, ema_momentum
FROM narratives
ORDER BY run_date DESC, ema_momentum DESC;
```

## Deleting a run

To remove all data for a specific date and allow the pipeline to re-run for that date, use `delete_run.py`:

```bash
# Delete a specific date
python delete_run.py 2025-01-15

# Interactive mode (defaults to today)
python delete_run.py
```

The script deletes in dependency order — `arguments` and `narratives` first (child tables), then `daily_runs` (parent table) — to respect the foreign key constraints:

```python
# delete_run.py
cursor.execute("DELETE FROM arguments WHERE run_date = ?", (run_date_str,))
cursor.execute("DELETE FROM narratives WHERE run_date = ?", (run_date_str,))
cursor.execute("DELETE FROM daily_runs WHERE run_date = ?", (run_date_str,))
```

<Warning>
  `delete_run.py` does not delete from `portfolio_history`. If you want to remove portfolio records for the date as well, run the following in the SQLite CLI before deleting the run:

  ```sql
  DELETE FROM portfolio_history WHERE run_date = '2025-01-15';
  ```

  Delete `portfolio_history` rows before the `daily_runs` row to respect the foreign key constraint.
</Warning>

## Full table creation SQL

This is the complete `init_db()` schema as defined in `data_manager/db.py`:

```python
# data_manager/db.py
def init_db(db_path: str = DB_PATH):
    """Crea las tablas si no existen."""
    conn = _get_connection(db_path)
    cursor = conn.cursor()

    cursor.executescript("""
        CREATE TABLE IF NOT EXISTS daily_runs (
            id              INTEGER PRIMARY KEY AUTOINCREMENT,
            run_date        TEXT NOT NULL UNIQUE,
            universe_size   INTEGER NOT NULL,
            total_arguments INTEGER NOT NULL,
            k_clusters      INTEGER NOT NULL,
            created_at      TEXT NOT NULL DEFAULT (datetime('now'))
        );

        CREATE TABLE IF NOT EXISTS arguments (
            id              INTEGER PRIMARY KEY AUTOINCREMENT,
            run_date        TEXT NOT NULL,
            ticker          TEXT NOT NULL,
            cluster_id      INTEGER,
            polarity        INTEGER NOT NULL,
            justification   TEXT NOT NULL,
            evidence        TEXT,
            probability     REAL,
            embedding_text  TEXT,
            FOREIGN KEY (run_date) REFERENCES daily_runs(run_date)
        );

        CREATE TABLE IF NOT EXISTS narratives (
            id              INTEGER PRIMARY KEY AUTOINCREMENT,
            run_date        TEXT NOT NULL,
            cluster_id      INTEGER NOT NULL,
            centroid_json   TEXT NOT NULL,
            avg_return      REAL DEFAULT 0.0,
            ema_momentum    REAL DEFAULT 0.0,
            n_arguments     INTEGER DEFAULT 0,
            FOREIGN KEY (run_date) REFERENCES daily_runs(run_date),
            UNIQUE(run_date, cluster_id)
        );

        CREATE TABLE IF NOT EXISTS portfolio_history (
            id              INTEGER PRIMARY KEY AUTOINCREMENT,
            run_date        TEXT NOT NULL,
            ticker          TEXT NOT NULL,
            side            TEXT NOT NULL CHECK(side IN ('LONG', 'SHORT')),
            rank            INTEGER NOT NULL,
            signal          REAL NOT NULL,
            weight          REAL NOT NULL,
            created_at      TEXT NOT NULL DEFAULT (datetime('now')),
            FOREIGN KEY (run_date) REFERENCES daily_runs(run_date)
        );

        CREATE INDEX IF NOT EXISTS idx_arguments_date ON arguments(run_date);
        CREATE INDEX IF NOT EXISTS idx_arguments_ticker ON arguments(ticker);
        CREATE INDEX IF NOT EXISTS idx_narratives_date ON narratives(run_date);
        CREATE INDEX IF NOT EXISTS idx_portfolio_date ON portfolio_history(run_date);
        CREATE INDEX IF NOT EXISTS idx_portfolio_side ON portfolio_history(side);
    """)

    conn.commit()
    conn.close()
```

## Migrating to PostgreSQL

`data_manager/db.py` is designed for a straightforward migration path. The comment at the top of the file states the intent explicitly:

```python
# data_manager/db.py
"""
Diseñado con SQL puro parametrizado para facilitar migración a PostgreSQL.
Solo cambiarías: sqlite3 → psycopg2/asyncpg y los tipos de columna.
"""
```

The required changes are minimal:

| Item | SQLite | PostgreSQL |
|---|---|---|
| Driver import | `import sqlite3` | `import psycopg2` or `import asyncpg` |
| Connection call | `sqlite3.connect(db_path)` | `psycopg2.connect(dsn=...)` |
| `TEXT` for dates | `TEXT NOT NULL` | `DATE NOT NULL` or `TIMESTAMPTZ` |
| `INTEGER PRIMARY KEY AUTOINCREMENT` | as-is | `SERIAL PRIMARY KEY` or `BIGSERIAL` |
| Placeholder style | `?` | `%s` |
| `executescript()` | built-in | Use `cursor.execute()` per statement |
| WAL pragma | `PRAGMA journal_mode=WAL` | Remove (not applicable) |

All SQL queries in `db.py` are parameterized and use standard SQL syntax, so they will work in PostgreSQL with only the placeholder character changed from `?` to `%s`.
