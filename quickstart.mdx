## Prerequisites

Before you begin, make sure you have:

- **Python 3.10 or higher** — the pipeline uses `match` statements and modern type hints
- An **OpenAI API key** — used for argument extraction (`gpt-4o`, `gpt-4o-mini`) and embeddings (`text-embedding-3-small`)
- A **NewsAPI key** — used to fetch recent news for each ticker ([newsapi.org](https://newsapi.org))

<Warning>
  The free tier of NewsAPI is limited to **100 requests per day**. Each ticker in your active universe consumes one request. The default `ACTIVE_UNIVERSE_LIMIT = 10` in `config.py` stays well within this limit. Do not increase it beyond 100 without a paid NewsAPI plan.
</Warning>

<Warning>
  OpenAI charges per token. Running `gpt-4o` over news articles for every ticker adds up quickly. At the default limit of 10 tickers, costs are minimal. At 500 tickers (full S&P 500), daily runs will generate a material monthly bill. See [/configuration/parameters](/configuration/parameters) before scaling up.
</Warning>

## Set up and run the pipeline

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/EconomiaUNMSM/MEME.git
    cd MEME
    ```
  </Step>

  <Step title="Create and activate a virtual environment">
    The automation scripts (`run_meme.bat`) expect the Python executable at `venv/Scripts/python`. Create the virtual environment inside the project root.

    ```bash
    # Create the virtual environment
    python -m venv venv

    # Activate it (Windows)
    venv\Scripts\activate

    # Activate it (macOS / Linux)
    source venv/bin/activate
    ```

    <Note>
      You must activate the virtual environment before installing dependencies or running the pipeline manually. Your prompt should show `(venv)` when it is active.
    </Note>
  </Step>

  <Step title="Install dependencies">
    ```bash
    pip install -r requirements.txt
    ```

    This installs: `yfinance`, `pandas`, `ta`, `requests`, `openai`, `python-dotenv`, `scikit-learn`, `scipy`, `langgraph`, `langchain-openai`, `lxml`, and `html5lib`.
  </Step>

  <Step title="Configure your .env file">
    Copy `.env.example` to `.env` in the project root and fill in your API keys:

    ```bash
    cp .env.example .env
    ```

    Then open `.env` and set both values:

    ```env
    # OpenAI API Key for Argument Extraction and Cluster Generation
    OPENAI_API_KEY=your_openai_api_key_here

    # NewsAPI Key for real-time market event extraction (Free or Premium)
    NEWSAPI_KEY=your_newsapi_key_here
    ```

    <Tip>
      You can get a free NewsAPI key at [newsapi.org/register](https://newsapi.org/register). The free plan supports up to 100 requests/day, which covers up to 100 tickers per run.
    </Tip>
  </Step>

  <Step title="Run the pipeline">
    **Option A — Windows batch file (recommended for daily automation):**

    ```bash
    run_meme.bat
    ```

    This activates the virtual environment and runs `main.py` in one step, then appends a timestamped entry to `meme_logs.txt`.

    **Option B — Direct Python invocation:**

    ```bash
    python main.py
    ```

    Use this option on macOS/Linux or when you have already activated the virtual environment.

    <Note>
      On first run, the pipeline fetches the current S&P 500 ticker list from Wikipedia. If that request fails, it falls back to a hardcoded list of 10 large-cap tickers: `AAPL`, `MSFT`, `NVDA`, `GOOGL`, `AMZN`, `META`, `TSLA`, `BRK-B`, `LLY`, `AVGO`.
    </Note>
  </Step>
</Steps>

## Default universe size

`config.py` sets `ACTIVE_UNIVERSE_LIMIT = 10` by default. This means only the first 10 tickers from the S&P 500 list are processed per run. This is intentional — it keeps costs low and requests within the free NewsAPI quota while you verify the pipeline is working correctly.

Once the pipeline is running cleanly end-to-end, you can increase this value. See [/configuration/parameters](/configuration/parameters) for all tunable settings and their cost/performance trade-offs.

## Automate daily runs (Windows)

To run the pipeline automatically every weekday at 18:00 Peru time (one and a half hours after the NYSE close), run `setup_schedule.bat` once as Administrator:

```bash
setup_schedule.bat
```

This registers a Windows Task Scheduler job named `MEME_DailyRun` that calls `run_meme.bat` daily at 18:00. To verify or remove it:

```bash
# Verify the task exists
schtasks /query /tn "MEME_DailyRun"

# Remove the task
schtasks /delete /tn "MEME_DailyRun" /f
```

<Warning>
  The machine must be powered on and connected to the internet at 18:00 each day for the scheduled task to run. Missing a day creates a gap in the EMA momentum history stored in `meme_history.db`.
</Warning>
